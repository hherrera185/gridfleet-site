// HOW IT WORKS

Built for operators
who care about output.

Three principles separate GridFleet from the demo-AI noise.

001 // ROUTING

Tasks land where they belong.

The fleet is an air-traffic controller for agentic work. Jobs are routed to the right agent with the right tools, in the right order, with no hand-off ambiguity.

002 // RECEIPTS

Every action leaves a trail.

Inputs, outputs, costs, and confidence scores are written to the ledger. You can answer "what did the fleet do, and why" without reading 200 logs.

003 // AUTONOMY

Self-healing, not self-running.

Supervisors restart wedged agents, re-queue stuck jobs, and surface drift before it costs you. You stay in command. The fleet stays in motion.

// ARCHITECTURE

Inputs land, work routes,
receipts settle.

Every job follows the same three-stage flow. Audit-grade by design — there is no path from input to outcome that skips the ledger.

// WHAT RUNS ON THE FLEET

Capabilities,
already in production.

Six agent classes, each tuned for a different surface of operational work. All running on the same control plane, all writing to the same ledger.

01
Forecasting
Probability-calibrated forecasts on Kalshi markets, weather grids, and options pricing. Confidence-scored outputs, costed per call.
02
Browser-as-a-service
Headed and headless sessions with in-house captcha solving, dwell jitter, and Phase-3 popup cascades. Per-action billing, full session receipts.
03
Code review & auto-impl
Branch-isolated agent dispatches that propose diffs, run sanity checks, and post structured reviews. No auto-merge — human-in-loop by design.
04
Self-healing ops
Supervisors that restart wedged services, sweep stale state, and surface drift before pager. Cron-guarded, TTL-locked, heartbeat-tracked.
05
Compliance audit
Continuous scans for secret leakage, policy drift, and config integrity. Quarantines prompt-injection rules. Vault rotation tracked end-to-end.
06
Strategic outreach
Agent-curated reply pipelines on social surfaces, with quality gates, sanitizers, and visual generation. Engagement is observed, not optimized blindly.
