How GridFleet differs

vs. single-LLM APIs (OpenAI, Anthropic, Google)

Single-LLM APIs return one model's completion to one prompt. GridFleet returns the result of a coordinated fleet operation, with receipts and audit trails as first-class outputs. You don't pick the model; the fleet routes the work to whichever agents it needs.

vs. browser-automation services (Browserbase, Browser-Use Cloud, Steel)

GridFleet ships an in-house captcha solver (Tesseract + Sonnet vision; no third-party captcha-solving fees), Phase 2a-2d behavioral realism, and Phase 3 multi-strategy popup dismissal. The browser pipeline is bundled with the rest of the fleet, so a single key gets you forecasts, screeners, and court feeds in addition to browse.

vs. DeFi research firms

The audit product covers the operational config that changes between formal audits — admin keys, oracle source lists, timelock bypass paths, governance drift — with sources cited per finding. Most "DeFi due diligence" stops at TVL and audit-firm logos.

Constraints we keep

  1. No public-cloud lock-in. The fleet runs on dedicated hardware. Cloudflare and Tailscale handle public exposure but the agents themselves are owner-operated.
  2. Receipts on every call. Every API call returns a request_id and writes a billing-events record. Audit trails are not optional.
  3. Reversible by default. Anything irreversible (capital, legal, hard-to-undo infra) requires human signoff. The fleet has agency, not authority.
  4. No data resale. Customer request bodies and response payloads are not stored beyond what's needed for the call.
